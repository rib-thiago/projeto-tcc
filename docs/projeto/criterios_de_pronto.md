# Criterios de Pronto

## 1. Objetivo

Este documento consolida criterios minimos de pronto para artefatos,
iteracoes e entregas relevantes do projeto.

Seu papel e reduzir ambiguidade sobre quando algo esta maduro o suficiente para
ser considerado utilizavel, revisavel ou encerravel.

Os gates e criterios de passagem entre blocos do processo permanecem definidos
em `docs/projeto/fases_do_processo.md`.

## 2. Definicao geral de pronto

Um item de trabalho pode ser considerado pronto quando, de forma proporcional ao
seu tipo e impacto:

- o objetivo declarado foi atingido
- o artefato principal esperado ficou utilizavel
- ha rastreabilidade suficiente da decisao ou entrega
- os impactos documentais relevantes foram tratados
- nao permanecem lacunas invisiveis que comprometam o proximo passo

## 3. Pronto por tipo de entrega

### 3.1 Documento estrutural

Considera-se pronto quando:

- o papel do documento esta claro
- o conteudo esta coerente com os documentos vizinhos
- nao ha contradicoes centrais conhecidas
- o documento pode ser usado como referencia real, e nao apenas como rascunho

### 3.2 Documento operacional

Considera-se pronto quando:

- o guia responde a perguntas praticas reais do trabalho cotidiano
- nao duplica desnecessariamente documentos estruturais
- orienta acao com clareza suficiente

### 3.3 Iteracao

Considera-se pronta quando:

- o objetivo principal da iteracao foi atingido ou explicitamente replanejado
- ha registro suficiente do que foi produzido, decidido ou aprendido
- a proxima iteracao ficou mais clara

### 3.4 Issue estrutural

Considera-se pronta quando:

- o objetivo declarado da issue foi atendido com evidencias documentais
- os documentos principais impactados foram revisados de forma cruzada
- ha base suficiente para mover a discussao para revisao formal ou proxima
  frente de trabalho

## 4. Uso recomendado

Este documento deve ser consultado:

- ao fim de iteracoes relevantes
- antes de encerrar uma issue estrutural
- quando houver duvida se algo esta pronto ou apenas avancado
- durante revisoes de coerencia entre backlog, documentos e execucao
